```bash
npm run format

```

The server runs on port 3000. `public/js/app.js` is generated by esbuild and is gitignored.

## Architecture

Two-part TypeScript app: an Express server (`src/server.ts`) and a browser client (`src/client/app.ts`), served as static files from `public/`.

**Server** (`src/server.ts`): Handles audio ingestion, processing, and serving.
- `POST /api/download` — downloads audio from a YouTube URL via `yt-dlp` (shell command)
- `POST /api/upload` — accepts local audio file via `multer`
- `GET /api/audio/:id` — streams the raw audio file for waveform rendering
- `POST /api/process` — trims audio to a selection and encodes to `.m4r` (iPhone ringtone) using `fluent-ffmpeg`; enforces 30-second iOS limit
- `GET /api/download-ringtone/:id` — serves the processed `.m4r` file

All temp files live in `tmp/` (gitignored). UUIDs are validated via regex before file path construction.

**Client** (`src/client/app.ts`): Vanilla TypeScript compiled via esbuild into `public/js/app.js`.
- Uses WaveSurfer.js with its RegionsPlugin for interactive waveform and drag-to-select region
- Handles YouTube URL download and local file upload flows
- Sends the selected start/end time + fade-out duration to `/api/process`, then triggers browser download of the `.m4r`

**Build split**: Server uses `tsconfig.json` (commonjs, targets Node), client uses `tsconfig.client.json` (ESM, targets browser DOM). Both are checked by `npm run typecheck`.

## External Dependencies

- `yt-dlp` must be installed on the system (not an npm package) for YouTube downloads
- `ffmpeg` must be installed on the system for audio processing
- Tailwind CSS is loaded via CDN in `public/index.html`
